📋 汇报摘要

本次技术方案汇报由张工主讲，介绍了新系统的微服务架构设计方案。李总及团队成员对关键技术选型进行了深入讨论。

🔑 关键决策

  • 采用微服务架构，按 DDD 划分 4 个核心服务

  • 每个微服务独立数据库，跨服务一致性采用 Saga 模式
  • API 网关统一使用 Kong
