## 完成事项

### 1. DashScope ASR 集成（Paraformer-v2）

**问题：** DashScope `Transcription.async_call()` 要求传入 HTTP 可访问的 URL，但 `dashscope.Files.upload()` 仅返回内部 file ID（`dashscope://file-xxx`），Transcription API 不支持该协议，导致 `FILE_DOWNLOAD_FAILED`。

**探索过程（5 轮迭代）：**

| # | 方案 | 结果 | 失败原因 |
|---|------|------|---------|
| 1 | `getPolicy` API 上传 + REST API 调用 | `InvalidParameter.ParseError` | REST API 参数格式不同于 SDK |
| 2 | `getPolicy` 上传 + SDK `Transcription.async_call()` | `FILE_403_FORBIDDEN` | 上传的 OSS 文件无公开读权限 |
| 3 | SDK `OssUtils.upload()` + SDK `Transcription.async_call()` | `FILE_DOWNLOAD_FAILED` | SDK 不自动添加 `X-DashScope-OssResourceResolve` header |
| 4 | 手动构造 `oss://` URL（含 bucket 名） + REST API | `FILE_DOWNLOAD_FAILED` | `oss://` URL 格式错误（不应含 bucket 名） |
| 5 | SDK `OssUtils.upload()` + REST API + `X-DashScope-OssResourceResolve: enable` | **成功** | — |

**最终方案：**
- **上传：** `dashscope.utils.oss_utils.OssUtils.upload()` → 返回 `oss://dashscope-instant/...` 格式 URL
- **转录：** REST API `POST /api/v1/services/audio/asr/transcription`，携带 `X-DashScope-Async: enable` 和 `X-DashScope-OssResourceResolve: enable` headers
- **轮询：** `GET /api/v1/tasks/{task_id}` 每 5 秒检查状态

**关键发现：** 通过 `inspect.getsource(OssUtils.upload)` 在容器内读取 SDK 源码，确认其返回格式为 `"oss://" + form_data["key"]`（不含 bucket 名），这与手动拼接的 URL 格式不同。

**修改文件：** `backend/app/services/asr.py`（完全重写）

### 4. 录音详情页修复

**问题 1：详情页空白**
- **原因：** 前端 `getRecordingDetail()` 请求 `GET /recordings/{id}?include=transcript,document`，但后端 API 不支持 `include` 参数
- **修复：** 改为 3 个并行请求（`Promise.allSettled`）分别获取 recording、transcript、document
- **文件：** `frontend/src/lib/api.ts`

**问题 2：DocumentPanel 无法渲染内容**
- **原因：** 硬编码了 `key_decisions`/`qa_records`/`action_items` 等字段名，但 LLM 实际返回 `key_info`/`next_steps`/`commitments`
- **修复：** 完全重写 DocumentPanel，使用 `SECTION_LABELS` 映射 + 通用 `renderSection()` 函数，动态支持所有 6 种场景类型的字段
- **文件：** `frontend/src/components/DocumentPanel.tsx`（完全重写）

---

### 5. 音频播放 + 转录文本联动

**问题 1：音频无法播放**
- **原因：** 前端（:3000）无法访问后端（:8000）的 `/uploads/*` 路径
- **修复：** 在 `next.config.ts` 添加 `/uploads/:path*` 代理规则
- **文件：** `frontend/next.config.ts`

**问题 2：file_url 路径不匹配**
- **原因：** 数据库存储的是容器内路径 `/app/uploads/...`，前端需要 `/uploads/...`
- **修复：** 在页面组件中添加 `.replace(/^\/app\/uploads\//, "/uploads/")` 路径转换
- **文件：** `frontend/src/app/recordings/[id]/page.tsx`

**问题 3：WaveSurfer.js 加载缓慢**
- **原因：** WaveSurfer 需要完整下载音频文件后才能解码生成波形，22MB 文件等待时间过长
- **修复：** 替换为原生 HTML5 `<audio>` 元素 + `preload="metadata"`，实现即时播放
- **新功能：**
  - 自定义进度条（带缓冲进度指示器）
  - 点击进度条跳转
  - `requestAnimationFrame` 平滑时间同步
  - 变速播放（0.5x–2.0x）
  - ±10 秒快进/后退
  - 转录文本段落点击 → 音频跳转到对应时间点
- **文件：** `frontend/src/components/AudioPlayer.tsx`（完全重写）
