# ListenWise 产品设计文档

> 智能录音转文档 Web 应用，将工作、生活、学习中的录音自动转录并生成场景化结构文档。

## 产品定位

**一句话描述：** 面向个人用户的智能录音转文档平台，不只是"语音转文字"，而是"录音变成可用的知识文档"。

**目标用户：**
- 职场人士：频繁参加各类会议，需要整理纪要
- 学生/学习者：录课程、讲座，需要学习笔记
- 家长：家长会、与老师沟通的记录需要留档
- 任何需要把"听过的内容"变成"可回顾文档"的人

**核心价值：** 用户上传一段会议录音，拿到的不是一堆原始文字，而是一份格式规范的会议纪要。

## 核心功能

### 1. 录音上传与录制

- **文件上传：** 支持 mp3、m4a、wav、mp4、webm 等主流格式，单文件最大支持 2 小时时长，批量上传每次最多 5 个
- **Web 端录音：** 浏览器内直接录音，适合临时场景
- **在线会议导入：** 支持上传腾讯会议、飞书、Zoom 等导出的音视频文件（提取音轨处理）

### 2. 智能转录

- **ASR 引擎：** 阿里云 Paraformer，中文识别准确率高
- **说话人分离：** 自动识别不同发言人，标注"发言人1""发言人2"等
- **标点与分段：** 自动恢复标点、智能分段，提升可读性
- **时间戳对齐：** 每段文字关联对应的音频时间点，支持点击文字跳转回听

### 3. 场景化模板输出

上传时选择场景（或系统自动识别），生成对应格式的文档：

| 场景 | 输出模板 |
|------|---------|
| 需求评审会 | 需求清单 + 讨论要点 + 决策结论 + 待办事项 |
| 汇报会议 | 汇报摘要 + 关键数据 + 问答记录 + 后续行动 |
| 领导大会 | 核心讲话要点 + 政策/方向提炼 + 金句摘录 |
| 家长会 | 老师反馈要点 + 学习建议 + 家长待办 |
| 电话录音 | 通话摘要 + 关键信息提取 + 承诺事项 |
| 学习录音 | 知识点大纲 + 重点笔记 + 概念解释 |

### 4. 录音管理与组织

- **场景分类：** 预设「工作会议」「学习笔记」「生活记录」「电话通话」四大类，用户可自定义新增
- **时间线视图：** 按日/周/月展示录音记录，直观回顾
- **文件夹 + 标签：** 用户自建文件夹，自由打标签，支持多标签筛选
- **全文搜索：** 在所有转录文本中搜索关键词
- **音文联动：** 点击转录文本中任意一句，跳转到对应音频位置回听原声
- **导出能力：** 支持导出为 Markdown、Word、PDF 格式

## 页面结构

| 页面 | 说明 |
|------|------|
| 仪表盘 | 统计卡片、处理中任务进度、最近录音列表、快捷操作、标签云 |
| 上传录音 | 拖拽上传/选择文件/Web 录音，选择场景类型，补充标题/标签/文件夹 |
| 录音详情 | 音频波形播放器 + 转录文本（音文联动） + 场景化文档结果（通过列表点击进入，非顶部导航入口） |
| 录音库 | 时间线视图 / 文件夹视图切换，全文搜索，场景筛选，标签管理 |
| 设置 | 转录参数、场景模板管理、导出格式偏好、通知设置 |

## 技术架构

```
用户浏览器（Next.js + TailwindCSS）
  ↓
后端 API（Python FastAPI）
  ├── 文件存储（阿里云 OSS）
  ├── ASR 转录（阿里云 Paraformer）
  ├── 智能分析（大模型 API — Claude / 通义千问）
  ├── 任务队列（Celery + Redis）
  └── 数据库（PostgreSQL）
```

### 技术选型理由

| 组件 | 选型 | 理由 |
|------|------|------|
| 前端 | Next.js + TailwindCSS | SSR 提升首屏体验，TailwindCSS 快速开发 |
| 后端 | Python FastAPI | 与 ASR/AI 生态契合度高，异步性能好 |
| ASR | 阿里云 Paraformer | 中文识别准确率领先，支持说话人分离、标点恢复，国内合规，有免费额度 |
| 智能分析 | 大模型 API | 负责场景化模板填充、摘要生成、关键信息提取 |
| 文件存储 | 阿里云 OSS | 存放音频原始文件，成本低 |
| 数据库 | PostgreSQL | 存储元数据、转录结果、用户数据 |
| 任务队列 | Celery + Redis | 长录音异步处理，转录完成后通知用户 |

### 核心流程

```
上传录音 → 选择场景 → 文件存入 OSS
→ Celery 任务：调用 Paraformer ASR 转录（显示进度）
→ 转录完成 → Celery 任务：调用大模型生成场景化文档
→ 结果写入 PostgreSQL → 通知用户
→ 用户查看/编辑 → 导出或归档
```

## MVP 边界

### 做的事
- 文件上传（多格式兼容）
- Web 端录音
- ASR 转录（说话人分离、标点恢复、时间戳）
- 6 种场景化模板输出
- 音文联动播放
- 录音管理（场景分类 + 时间线 + 文件夹标签）
- 全文搜索
- 导出 Markdown / Word / PDF

### 不做的事
- 团队协作和权限管理
- 实时转录（流式识别）
- 多语言混合识别
- 移动端适配（后续迭代）
- 自定义 ASR 模型训练
- 商业化计费系统

## 数据模型（核心表）

```
User
├── id, name, email, created_at

Recording
├── id, user_id, title, file_url (OSS)
├── scene_type, folder_id, duration, file_size
├── status (uploading / transcribing / analyzing / done / failed)
├── created_at, updated_at

Transcript
├── id, recording_id
├── segments (JSON: [{start, end, speaker, text}, ...])
├── full_text, word_count

Document
├── id, recording_id, scene_type
├── content (JSON: 场景化结构数据)
├── format_version

Folder
├── id, user_id, name, parent_id

Tag
├── id, user_id, name

RecordingTag (多对多)
├── recording_id, tag_id
```

## 后续迭代方向

1. **桌面端应用** — Electron / Tauri 封装，本地文件管理更便捷
2. **移动端适配** — 响应式或独立 App，手机录音即时处理
3. **实时转录** — 流式 ASR，边录边转
4. **自动场景识别** — 根据录音内容自动判断场景类型
5. **团队版** — 协作、权限、会议纪要分发
6. **AI 对话** — 基于转录内容做问答，"这次会议谁提出了什么？"
