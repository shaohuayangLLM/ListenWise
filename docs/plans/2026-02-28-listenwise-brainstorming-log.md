## 1. 头脑风暴阶段

### 1.1 需求起源

**用户原始描述：**

> 工作中需求评审会议，汇报会议，领导大会，生活中家长会，重要的电话录音，以及学习的一些录音文件，这些都需要进行转录为文档，想做一个产品，可以将这些杂乱的录音进行处理，整理和分析。

**核心痛点提炼：**
- 录音文件来源多样（工作/生活/学习），缺乏统一处理方案
- 手动转录和整理效率低
- 不同场景的录音需要不同格式的文档输出

### 1.2 问答决策过程

通过逐个问题的形式明确需求方向（使用 superpowers:brainstorming 技能）：

| # | 问题 | 用户选择 | 补充说明 |
|---|------|---------|---------|
| 1 | 产品形态 | **Web 应用** | "先做 Web 应用吧，后面再做桌面" |
| 2 | 目标用户 | **个人用户为主** | 选择选项 1 |
| 3 | 录音来源 | **全部格式** | "上述都有" — 手机录音、在线会议导出、混合来源、Web 录音 |
| 4 | 输出形式 | **转录 + 整理 + 场景化模板** | 选择选项 3 — 不只是文字，还要按场景生成结构化文档 |
| 5 | ASR 引擎 | **阿里云 Paraformer** | "看你的建议" — AI 推荐 Paraformer（中文识别准确率高、国内合规） |
| 6 | 商业化 | **先不考虑** | 选择选项 4 |
| 7 | 组织方式 | **场景 + 时间线 + 标签文件夹** | 多维度组合 |
| 8 | 时长限制 | **1-2 小时** | 选择选项 2 |

## 2. 原型设计阶段

### 2.1 原型制作过程

使用 `/frontend-design` 技能创建交互式 HTML 原型。

**文件位置：** `docs/prototypes/wireframes.html`

**初始版本：**
- 5 个页面：仪表盘、上传录音、录音详情、录音库、设置
- 采用深色主题（暗色背景）
- 产品名称：声纹 VoiceDoc

### 2.2 迭代修改记录

#### 迭代 1：色调调整

**用户反馈：** "色条改成浅白色"

**修改：** 将整体色调从深色暗色主题改为浅白色明亮主题

| 变量 | 修改前（深色） | 修改后（浅色） |
|------|---------------|---------------|
| --bg | #0a0b14 | #f5f6fa |
| --surface | #12131f | #ffffff |
| --text | #e8e9f0 | #1a1d2e |
| --border | #1e2033 | #e0e2eb |

**用户反馈：** "效果很好"

#### 迭代 2：导航结构调整

**用户反馈：** "顶部的结果tab名称改下：仪表盘、上传录音、录音库和设置，录音详情暂不做为一级入口"

**修改：**
- 顶部导航从 5 个 Tab 缩减为 4 个
- 移除「录音详情」Tab（改为从列表点击进入）
- 最终导航结构：仪表盘 → 上传录音 → 录音库 → 设置

#### 迭代 3：产品名称变更

**用户反馈：** "产品名称也改为ListenWise，包括项目名称以及各类产物都改成这个名称"

**修改：**
- 产品名称：声纹 VoiceDoc → ListenWise
- 项目目录：`/Users/ysh/Manual Library/ClaudeCode/录音文件处理` → `/Users/ysh/Manual Library/ClaudeCode/ListenWise`
- 更新所有文档和原型中的名称引用

### 2.3 最终原型确认

**用户反馈：** "可以了，这个原型很好的，继续后面的内容"

**最终原型特征：**
- 浅白色明亮主题
- 品牌紫 #6c5ce7 作为主色调
- 4 个顶部导航 Tab（仪表盘、上传录音、录音库、设置）
- 录音详情通过列表点击进入
- Noto Sans SC + JetBrains Mono 字体方案
- 毛玻璃效果导航栏 (backdrop-blur)

### 2.4 视觉风格关键决策

| 决策点 | 最终选择 | 理由 |
|--------|---------|------|
| 主题色调 | 浅白色 (light) | 用户明确要求 "色条改成浅白色" |
| 品牌色 | #6c5ce7 (紫) | 区别于常见蓝色系，辨识度高 |
| 场景分类色 | 紫/青/粉/橙 | 四大分类色彩对比鲜明 |
| 字体 | Noto Sans SC | 中文渲染优质，Google Fonts 免费 |
| 导航结构 | 4 个顶部 Tab | 用户要求精简，录音详情非一级入口 |
| 圆角 | 12px 卡片, 8px 按钮 | 柔和但不过度圆润 |
| 图标 | Lucide React | 线条风格统一，轻量 |

## 4. 实现阶段

### 4.1 实现方案选择

用户选择使用 **Agent Team** 进行实现，而非手动逐步执行。

**团队规模讨论：** 提供了 3 人/4 人/5 人/6 人四个方案，用户选择 **4 人团队**。

**团队分工：**

| 角色 | 代号 | 职责 |
|------|------|------|
| 团队 Lead | lead | 架构决策、数据库设计、任务分配、Docker 部署 |
| 后端 API | backend-api | FastAPI 路由、文件上传、搜索导出 |
| AI 服务 | ai-service | ASR 转录集成、LLM 文档生成、Celery 任务 |
| 前端工程 | frontend | Next.js 页面、组件开发、交互实现 |

### 4.2 开发监控

用户提出要使用 tmux 分屏监控各 Agent 状态：

> "开始，之前几次agentteam都没有tumx分屏，这次先确定下能不能分屏显示各个agent"

选择方案：**tmux 监控日志文件** — 每个 Agent 向各自的日志文件追加输出，tmux 4 格分屏使用 `tail -f` 实时显示。

**注意事项：** 初次启动 tmux 后用户在 IDE 终端中看不到，后续通过 `osascript` 打开新的 Terminal.app 窗口自动 `tmux attach`，用户确认 "看到了"。

### 4.3 实现计划

18 个任务，5 个阶段：

| 阶段 | 任务范围 | 说明 |
|------|---------|------|
| Phase 1 | Task 1-3 | 项目搭建（前后端初始化 + 数据库） |
| Phase 2 | Task 4-9 | 核心流程（上传、转录、详情、仪表盘） |
| Phase 3 | Task 10-14 | 管理功能（录音库、搜索、设置、导出） |
| Phase 4 | Task 15-17 | 体验打磨（Web 录音、进度更新、空状态） |
| Phase 5 | Task 18 | Docker 化部署 |

### 4.4 开发过程中的问题与解决

| 问题 | 原因 | 解决方案 |
|------|------|---------|
| PostgreSQL 未安装 | 开发机上没有 PostgreSQL | `brew install postgresql@16` |
| psycopg2 缺失 | Alembic 需要同步驱动 | `uv pip install psycopg2-binary` |
| 数据库连接失败 | 用户名错误 (postgres → ysh) | 修改 alembic.ini 连接字符串 |
| setuptools 包冲突 | uploads 目录被识别为包 | 添加 pyproject.toml 配置 |
| tmux 不可见 | IDE 终端非标准 Terminal | osascript 打开新 Terminal 窗口 |
