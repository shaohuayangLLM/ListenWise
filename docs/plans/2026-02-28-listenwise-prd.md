## 1. 产品概述

### 1.1 背景与痛点

工作中的需求评审、汇报会议、领导大会，生活中的家长会、重要电话录音，学习中的课程录音——人们日常积累了大量录音文件，但这些录音存在以下痛点：

1. **回听成本高** — 一段 1 小时的录音，要找到关键信息必须从头听到尾
2. **整理耗时** — 手动将录音整理为会议纪要、学习笔记，耗费大量时间
3. **格式混乱** — 不同场景的录音缺乏统一的整理规范和输出格式
4. **管理困难** — 录音文件散落在手机、电脑各处，难以检索和回顾
5. **价值流失** — 大量有价值的信息因为没有被及时整理而遗忘

### 1.2 产品定义

ListenWise 是一个面向个人用户的智能录音转文档 Web 应用。用户上传录音文件后，系统自动完成语音转文字，并根据录音的使用场景（会议、学习、通话等）生成结构化的文档。

**核心理念：录音不只是转文字，而是变成可用的知识文档。**

### 1.3 目标用户

| 用户类型 | 典型场景 | 核心诉求 |
|---------|---------|---------|
| 职场人士 | 需求评审、汇报会议、领导大会 | 快速生成会议纪要，提取决策和待办 |
| 学生/学习者 | 课程录音、讲座、培训 | 自动生成学习笔记，提炼知识点 |
| 家长 | 家长会、与老师沟通 | 记录反馈要点，不遗漏重要信息 |
| 通用个人用户 | 电话录音、访谈记录 | 留档归档，快速回顾关键内容 |

## 2. 功能需求

### 2.1 录音上传与录制

#### FR-2.1.1 文件上传

**描述：** 用户可以将本地录音/录像文件上传至系统进行处理。

**详细需求：**
- 支持拖拽上传和点击选择文件两种方式
- 支持格式：mp3、m4a、wav、mp4、webm、ogg、flac、aac
- 单文件时长限制：最长 2 小时
- 单文件大小限制：最大 500MB
- 批量上传：单次最多 5 个文件
- 视频文件自动提取音轨处理，不保留视频画面
- 上传过程显示进度条
- 支持断点续传（文件 > 50MB 时启用分片上传）

**验收标准：**
- [ ] 拖拽 mp3 文件到上传区域，文件开始上传并显示进度
- [ ] 上传 200MB 的 mp4 文件，系统自动提取音频并进入转录流程
- [ ] 同时选择 6 个文件上传，系统提示"单次最多上传 5 个文件"
- [ ] 上传超过 2 小时的录音文件，系统提示"单文件时长不超过 2 小时"

#### FR-2.1.2 Web 端录音

**描述：** 用户可以在浏览器中直接录音，录音完成后进入处理流程。

**详细需求：**
- 点击「开始录音」按钮，请求麦克风权限后开始录音
- 录音过程中显示时长计时和音量波形动画
- 支持暂停/继续录音
- 录音完成后自动保存为 webm 格式
- 录音时长限制：最长 2 小时
- 录音过程中如页面意外关闭，已录制内容自动保存为草稿

**验收标准：**
- [ ] 点击录音按钮，浏览器弹出麦克风授权请求
- [ ] 录音过程中可暂停和继续，时长计时准确
- [ ] 录音结束后文件自动进入上传处理流程

#### FR-2.1.3 场景选择

**描述：** 上传/录音时，用户选择录音所属的场景类型，系统据此生成对应格式的文档。

**详细需求：**
- 预设 6 种场景类型：
  1. 需求评审会
  2. 汇报会议
  3. 领导大会
  4. 家长会
  5. 电话录音
  6. 学习录音
- 场景选择为必选项，默认不选中
- 用户可在设置中自定义新增场景类型

**验收标准：**
- [ ] 上传时不选择场景，「开始转录」按钮置灰不可点击
- [ ] 选择「汇报会议」场景后上传，转录完成后生成汇报会议模板格式的文档

#### FR-2.1.4 补充信息

**描述：** 上传时用户可填写标题、标签、文件夹等补充信息。

**详细需求：**
- 录音标题：选填，不填则使用原始文件名
- 标签：选填，支持输入多个标签（逗号分隔），支持从已有标签中选择
- 文件夹：选填，下拉选择已有文件夹或新建
- 备注说明：选填，自由文本

**验收标准：**
- [ ] 不填任何补充信息也能正常上传和处理
- [ ] 填写标签"紧急, Q1评审"后上传，录音自动关联这两个标签

### 2.2 智能转录

#### FR-2.2.1 语音识别

**描述：** 系统使用 ASR 引擎将录音中的语音自动转换为文字。

**详细需求：**
- ASR 引擎：阿里云 Paraformer
- 支持中文普通话识别
- 自动恢复标点符号（逗号、句号、问号、感叹号等）
- 智能分段：根据语义和停顿自动划分段落
- 转录准确率目标：标准普通话 ≥ 95%

**验收标准：**
- [ ] 上传一段清晰的中文会议录音，转录文本可读且标点准确
- [ ] 转录结果自动分段，每段不超过 3-5 句话

#### FR-2.2.2 说话人分离

**描述：** 自动识别录音中的不同发言人并在转录文本中标注。

**详细需求：**
- 自动检测录音中的发言人数量（上限 10 人）
- 每段转录文本标注对应的发言人标识（"发言人 A""发言人 B"等）
- 用户可在详情页手动修改发言人名称（如将"发言人 A"改为"张工"）
- 发言人名称修改后全文同步更新

**验收标准：**
- [ ] 上传 3 人对话录音，系统正确识别出 3 位发言人
- [ ] 将"发言人 A"重命名为"张工"，全文所有该发言人的标注同步更新

#### FR-2.2.3 时间戳对齐

**描述：** 转录文本中每段话关联对应的音频时间点。

**详细需求：**
- 每段转录文本标注起始时间戳，格式 `MM:SS` 或 `HH:MM:SS`
- 时间戳精度：逐句级别
- 时间戳粒度可在设置中调整（逐句 / 每 30 秒 / 每分钟）

**验收标准：**
- [ ] 转录文本中每段话前显示时间戳
- [ ] 时间戳与实际音频位置偏差 ≤ 2 秒

#### FR-2.2.4 转录进度

**描述：** 录音处理过程中向用户展示实时进度。

**详细需求：**
- 状态流转：上传中 → 排队中 → 转录中 → 分析中 → 已完成
- 转录中状态显示百分比进度
- 仪表盘「处理中」区域展示所有进行中任务
- 转录完成后发送浏览器通知（需用户授权）

**验收标准：**
- [ ] 上传完成后仪表盘显示该录音状态为"排队中"
- [ ] 转录过程中进度百分比持续更新
- [ ] 转录完成后浏览器弹出通知提醒

### 2.3 场景化文档生成

#### FR-2.3.1 模板引擎

**描述：** 转录完成后，系统调用大模型根据场景类型生成结构化文档。

**详细需求：**

每种场景的输出结构：

**需求评审会：**
- 会议基本信息（时间、参会人）
- 需求清单（逐条列出讨论的需求项）
- 讨论要点（每个需求的关键讨论内容）
- 决策结论（达成一致的决定）
- 待办事项（明确责任人和截止时间）

**汇报会议：**
- 汇报摘要（200 字以内）
- 关键数据/指标
- 问答记录（问题 + 回答配对）
- 后续行动项

**领导大会：**
- 核心讲话要点（分条列出）
- 政策/方向提炼
- 金句摘录（原话引用）
- 关键数字/目标

**家长会：**
- 老师反馈要点
- 学科表现概况
- 学习建议
- 家长待办清单

**电话录音：**
- 通话摘要（100 字以内）
- 关键信息提取（人名、数字、日期等）
- 双方承诺事项
- 需要跟进的事项

**学习录音：**
- 知识点大纲（层级结构）
- 重点笔记（核心概念标粗）
- 概念解释（专业术语释义）
- 复习要点

**验收标准：**
- [ ] 上传一段需求评审录音并选择「需求评审会」，生成的文档包含需求清单、讨论要点、决策结论、待办事项四个板块
- [ ] 生成的文档内容准确反映录音实际讨论内容，无明显幻觉
- [ ] 待办事项中正确提取出责任人（如录音中有提及）

#### FR-2.3.2 文档编辑

**描述：** 用户可以在详情页对生成的文档进行编辑和修改。

**详细需求：**
- 支持对文档内容进行富文本编辑（标题、加粗、列表、引用）
- 编辑后自动保存
- 支持撤销/重做操作
- 可重新生成文档（使用原始转录文本重新调用大模型）

**验收标准：**
- [ ] 修改文档中的某条待办事项，刷新页面后修改内容保留
- [ ] 点击「重新生成」，系统重新调用大模型生成新的文档（覆盖用户编辑）

### 2.4 录音详情页

#### FR-2.4.1 音频播放器

**描述：** 详情页顶部提供音频播放器，支持波形可视化和多种播放控制。

**详细需求：**
- 显示音频波形图，已播放部分和未播放部分用不同颜色区分
- 点击波形任意位置跳转到该时间点
- 播放控制：播放/暂停、快进 15 秒、快退 15 秒
- 播放速度调节：0.5x / 0.75x / 1.0x / 1.25x / 1.5x / 2.0x
- 显示当前时间 / 总时长
- 底部显示发言人图例（颜色标识）

**验收标准：**
- [ ] 点击播放按钮，音频开始播放，波形进度同步推进
- [ ] 切换到 1.5x 速度，音频以 1.5 倍速播放
- [ ] 点击波形中间位置，音频跳转到对应时间点

#### FR-2.4.2 音文联动

**描述：** 转录文本与音频播放器联动，实现"听到哪看到哪，看到哪听到哪"。

**详细需求：**
- 音频播放时，当前正在播放的转录段落自动高亮
- 转录文本自动滚动跟随播放进度
- 点击任意一段转录文本，音频跳转到该段对应的时间点开始播放
- 高亮效果平滑过渡，不生硬跳动

**验收标准：**
- [ ] 播放音频时，对应的转录文本行自动高亮并滚动到可视区域
- [ ] 点击转录文本中 `05:32` 时间戳的段落，音频跳转到 5 分 32 秒处开始播放

#### FR-2.4.3 详情页布局

**描述：** 详情页采用左右分栏布局，左侧转录文本，右侧场景化文档。

**详细需求：**
- 顶部：录音标题 + 场景标签 + 操作按钮（编辑、导出、分享）
- 中部：音频播放器（波形 + 控制栏）
- 下方左侧：转录文本面板（带时间戳、发言人标注）
- 下方右侧：场景化文档面板（支持 tab 切换「场景纪要 / 原始摘要」）
- 转录文本面板显示总字数
- 详情页通过录音列表点击进入，不作为顶部导航入口

**验收标准：**
- [ ] 从仪表盘或录音库点击某条录音，进入详情页
- [ ] 左右面板可独立滚动，互不干扰
- [ ] 右侧面板可切换「场景纪要」和「原始摘要」两个 tab

### 2.5 录音库

#### FR-2.5.1 时间线视图

**描述：** 按时间倒序展示所有录音记录，以日为单位分组。

**详细需求：**
- 按日分组，每日显示该日期下的所有录音
- 每条记录展示：标题、上传时间、场景标签、时长、处理状态
- 左侧时间线连线视觉效果
- 支持按日/周/月粒度切换

**验收标准：**
- [ ] 打开录音库默认显示时间线视图
- [ ] 今天上传的 2 条录音显示在「今天」分组下
- [ ] 点击某条录音跳转到详情页

#### FR-2.5.2 文件夹视图

**描述：** 以文件夹层级结构展示和管理录音。

**详细需求：**
- 左侧文件夹树形导航，支持展开/折叠
- 预设顶级分类：全部文件、工作、学习、生活、电话通话
- 用户可创建自定义文件夹（支持嵌套，最深 3 层）
- 每个文件夹显示录音数量
- 右侧展示当前文件夹下的录音列表
- 支持拖拽录音到文件夹中移动
- 左侧底部展示常用标签，点击标签筛选

**验收标准：**
- [ ] 切换到文件夹视图，左侧显示文件夹树
- [ ] 点击「工作 > Q1 评审」文件夹，右侧只展示该文件夹下的录音
- [ ] 点击「+ 新建文件夹」可创建新文件夹

#### FR-2.5.3 搜索与筛选

**描述：** 支持全文搜索和多维度筛选。

**详细需求：**
- 搜索范围：录音标题、转录全文、标签名称
- 搜索结果高亮匹配关键词
- 场景类型筛选：顶部 filter chips，支持单选切换
- 标签筛选：支持多标签组合筛选

**验收标准：**
- [ ] 搜索"微服务架构"，返回转录文本中包含该词的所有录音
- [ ] 点击「学习笔记」filter chip，只展示学习类录音

### 2.6 仪表盘

#### FR-2.6.1 统计卡片

**描述：** 展示用户的录音使用数据概览。

**详细需求：**
- 总录音数（本月新增数量）
- 总时长（累计转录字数）
- 待处理数量（预计完成时间）
- 本周新增（较上周对比）

**验收标准：**
- [ ] 新用户首次访问，所有统计数字为 0
- [ ] 上传新录音后，总录音数和待处理数量实时更新

#### FR-2.6.2 处理中任务

**描述：** 展示当前正在处理中的录音任务及进度。

**详细需求：**
- 列出所有非「已完成」状态的录音
- 每条显示：文件名、时长、上传时间、进度条、状态文字
- 处理完成后自动从列表中移除

#### FR-2.6.3 最近录音

**描述：** 展示最近处理完成的录音列表。

**详细需求：**
- 默认展示最近 5 条已完成的录音
- 每条显示：标题、时间、时长、发言人数、场景标签
- 点击进入详情页
- 提供「查看全部」链接跳转到录音库

#### FR-2.6.4 快捷操作

**描述：** 右侧提供常用操作快捷入口。

**详细需求：**
- 4 个快捷按钮：上传文件、开始录音、搜索内容、待办事项
- 常用标签云展示
- 存储用量进度条

### 2.8 导出功能

#### FR-2.8.1 文档导出

**描述：** 用户可将场景化文档导出为多种文件格式。

**详细需求：**
- 支持格式：Markdown (.md)、Word (.docx)、PDF (.pdf)
- 导出内容可选：仅场景化文档 / 场景化文档 + 原始转录
- 时间戳可选包含或排除
- 导出文件命名规则：`{录音标题}_{场景类型}_{日期}.{格式}`
- 点击导出后直接下载到本地

**验收标准：**
- [ ] 点击导出 → 选择 Word 格式 → 浏览器下载 .docx 文件
- [ ] 导出的 Word 文件中格式正确，标题、列表、引用样式保留

## 3. 非功能需求

### 3.1 性能要求

| 指标 | 目标值 |
|------|--------|
| 页面首屏加载 | ≤ 2 秒 |

### 3.2 可靠性

- 转录任务失败时自动重试（最多 3 次）
- 失败后向用户展示失败原因和重试按钮
- 音频文件上传至 OSS 后保留原始文件，确保可重新处理
- 数据库每日自动备份

### 3.3 安全性

- 用户数据隔离，无法访问他人录音
- 音频文件存储使用私有 Bucket，通过签名 URL 访问
- 敏感信息（如手机号、身份证号）在转录文本中不做特殊处理（MVP 阶段）
- HTTPS 全站加密

## 6. 用户核心流程

### 6.1 新用户首次使用

```
访问 ListenWise → 注册/登录
→ 进入仪表盘（空状态引导）
→ 点击「上传录音」
→ 拖入文件 → 选择场景类型
→ 开始转录 → 等待处理完成
→ 进入详情页查看结果
→ 编辑/导出文档
```

### 6.2 日常使用流程

```
上传录音/开始浏览器录音
→ 选择场景 + 填写标题/标签（可选）
→ 提交处理 → 返回仪表盘查看进度
→ 收到完成通知 → 点击进入详情页
→ 播放回听 + 查看场景文档
→ 编辑修正 → 导出/归档
```

### 6.3 搜索回顾流程

```
打开录音库 → 输入关键词搜索
→ 找到目标录音 → 进入详情页
→ 点击匹配段落 → 音频跳转到对应位置回听
```
